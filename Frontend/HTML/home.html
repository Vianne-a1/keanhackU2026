Core Features

Feature 1

Policy Confusion Agent

Uses multi-policy reasoning instead of basic FAQ lookup. Example: software purchase requests are checked against procurement budget limits and IT security approved vendors before returning a verdict.

  • Semantic retrieval from company-specific policy docs
  • Chain-of-thought style internal reasoning steps
  • Outputs: Approved, Flagged, or Prohibited

Feature 2

Fraud & Fairness Detection

Audits uploaded vendor contracts for suspicious terms and unfair clauses.

  • Fraud red flags: ghost services, unusual payments, bank account changes
  • Fairness checks: predatory renewals, hidden fees, quote mismatches
  • Outputs: Safe, Warning, or Red Flag

System Flow

A. Management Flow (Company Side)

  1. Document Ingestion: Admins upload policies through a secure portal.
  2. Vectorization Engine: Documents are chunked and embedded for meaning-based retrieval.
  3. Organization Scoping: Strict isolation keeps Company X data inaccessible to Company Y.

B. Assistant Flow (User Side)

  1. Natural Language Query: Employees describe purchasing, data, or vendor scenarios.
  2. Semantic Search: Relevant policy passages are fetched for that employee organization.
  3. Policy Agent Verdict: Approved, Flagged, or Prohibited with plain-English explanation.

Usage Guide

  1. Admins upload policy documents and assign them to the correct organization scope.
  2. System vectorizes policy text and stores embeddings in a vector namespace.
  3. Users ask policy questions in natural language.
  4. Assistant returns verdict and explanation with supporting policy context.
  5. Optional: Upload vendor contracts for fraud and fairness red-flag analysis.
